## 🇰🇷 0. Korean Lyric Positioning (한국어 가사 포지셔닝) — v1.7 NEW

> **"이 프로젝트의 핵심 차별점은 한국어 가사 플레이리스트라는 점이다."**

대부분의 플레이리스트가 영어/Lo-fi 중심인 환경에서,
우리는 **우리말의 정서와 문장 밀도**를 전면에 둔다.

---

### 0.1 포지셔닝 원칙 (Why)

| 원칙 | 설명 |
|------|------|
| **의미 있는 가사** | 이 채널은 배경음악이 아니라 **의미가 있는 한국어 가사 음악**을 다룬다 |
| **혼자 읽히는 언어** | 가사는 "따라 부르기"보다 **혼자 읽히고, 혼자 들리는 언어**를 지향한다 |
| **소리가 먼저** | 감정은 과장하지 않고, **말보다 소리가 먼저 닿는** 한국어 문장을 목표로 한다 |

---

### 0.2 가사 설계 철학 (Lyric Philosophy)

**금지:**
- ❌ 설명적 문장
- ❌ 감정 직접 표출 (사랑해, 슬퍼, 외로워 등)
- ❌ 영어, 외래어, 메타 표현

**지향:**
- ✅ 사물·공간·현상 중심 묘사
- ✅ 한 문장 한 이미지 원칙
- ✅ **"가사 없는 음악처럼 들리지만, 읽으면 남는 가사"**

---

### 0.3 플레이리스트 맥락에서의 가사 규칙

| 규칙 | 설명 |
|------|------|
| **단독 완성도** | 영상 전체가 가사를 '보여주지' 않더라도, 각 곡은 단독으로 가사 완성도를 가져야 한다 |
| **시점 대변** | 가사는 플레이리스트의 분위기를 설명하는 수단이 아니라, **청자의 하루 특정 시점을 대신 말해주는 언어**여야 한다 |

---

### 0.4 한국어 검증 체크리스트 (필수)

가사/트랙 생성 시 아래 항목을 **반드시 자체 검증**:

| # | 질문 | PASS 조건 |
|---|------|----------|
| K1 | 이 가사는 **한국어여야만 성립**하는가? | 한국어 고유 정서/표현 포함 |
| K2 | 영어로 번역하면 **힘이 빠지는 지점**이 있는가? | 번역 불가능한 뉘앙스 존재 |
| K3 | 배경음악으로 흘러도 어색하지 않으면서, **읽으면 문장으로 남는가**? | 청각+독해 이중 만족 |

**FAIL 조건:**
- 3개 중 2개 이상 No → 가사 재작성

---

### 0.5 메타/브랜딩 적용 규칙

| 항목 | 규칙 |
|------|------|
| **영상 제목/설명** | "한국어"를 과도하게 전면 배치하지 않는다 |
| **내부 규칙** | 가이드와 내부 규칙에서 명확한 정체성으로 고정 |
| **장기 누적** | 해시태그, 소개 문구, 세계관 설정에서 "우리말 가사 플레이리스트"를 지속적으로 누적 강화 |

## 🎯 0.6 LYRICS.md의 존재 이유

VIBE-M에서 가사는 "감정 표현"이 아니라 **음악을 안정적으로 생성하기 위한 설계 데이터**다.

이 문서의 목표는 다음 3가지를 동시에 만족시키는 것이다.

1. Suno가 멜로디를 **안정적으로 복제/변주**하게 만든다.
2. 인간 리스너에게 **자연스럽고 트렌디한 감정선**을 전달한다.
3. 알고리즘이 **양산형/중복 콘텐츠**로 오인하지 않게 한다.

## 🧠 1. Core Lyrical Engineering Rules (절대 규칙)

### 1.1 Metric Mirroring (V1–V2 대칭)

- Verse 1과 Verse 2는:
  - **음절 수**
  - **행 수**
  - **문장 구조**
를 완전히 동일하게 유지한다.

목적:
- Suno가 Verse 1의 멜로디 컨투어를 Verse 2에 그대로 복제하게 유도
- 감정만 고조되고 멜로디는 흔들리지 않게 만들기 위함

---

### 1.2 Rhyme / Ending Mirroring (종지감 통일) — 강화

- V1과 V2 각 행의 끝 단어는 다음을 **엄격히** 일치시킨다.
  - **품사 (형용사 / 명사 / 동사)** — 가장 중요
  - 어미 패턴 (-져, -어, -에 등)
  - 종지감 길이

**품사 불일치 = FAIL:**
- ❌ V1 "서 있어" (동사) vs V2 "차가워" (형용사) → FAIL
- ✅ V1 "서 있어" (동사) vs V2 "얼어" (동사) → PASS
- ✅ V1 "차가워" (형용사) vs V2 "따가워" (형용사) → PASS

예시 (엄격 적용):
- `차가워 (adj)` → `따가워 (adj)` ✓
- `얼굴들 (noun)` → `흔적들 (noun)` ✓
- `지워져 (verb)` → `무너져 (verb)` ✓
- `서 있어 (verb+있다)` → `얼어 (verb)` ✓

목적:
- 멜로디 끝 처리(ending note)가 동일하게 재현되도록 강제
- **품사 불일치 시 AI가 다른 멜로디 패턴을 생성할 위험**

---

### 1.3 Chorus Static Rule (후렴 고정)

- 후렴 가사는:
  - **전 구간 100% 동일**
  - 3~4행
  - 각 행 6~10음절 권장

Chorus 2, Chorus 3에서도:
- 가사는 절대 바꾸지 않는다.
- 감정 증폭은 **사운드와 보컬 레이어**로만 처리한다.

---

### 1.4 Bridge Anchor Rule (테제 + 장면)

Bridge는 반드시 아래 구조를 따른다.

- Thesis (고정 1라인)
  - 곡의 철학적 중심 문장
  - 보통 마지막 행
- Scene (2~3라인)
  - 설명 ❌
  - 감정 설명 ❌
  - **구체적인 동작 / 감각 / 이미지**만 허용

추가 규칙:
- Bridge 1과 Bridge 2는
  - Thesis는 동일
  - Scene 라인은 최소 2행 이상 변주

---

### 1.5 Vocabulary Independence (어휘 독립성)

- 같은 플레이리스트 내 **트랙 간 핵심 은유(메타포)가 겹치면 안 된다**.
- 각 트랙은 독립적인 "이미지 축"을 가져야 한다.

규칙:
- **제목에 사용된 단어**는 다른 트랙에서 핵심 키워드로 사용 금지
- 같은 감각 계열도 2트랙 이상 연속 사용 금지 (예: 소리→소리→소리 ❌)

예시:
- Track 03 제목이 "잔향"이면 → Track 04에서 "잔향/메아리/울림" 금지
- Track 02가 "습기/유리/김" 축이면 → Track 03은 다른 축 (그림자/시간/공간 등)

목적:
- 알고리즘이 "같은 곡 반복"으로 오인하는 것 방지
- 플레이리스트 전체의 다양성과 독립성 확보

---

### 1.6 Snapshot Hook Rule (스냅샷 훅)

- Chorus(후렴) 훅은 **설명형**이 아닌 **이미지형**이어야 한다.
- 4행 중 최소 1~2행은 "명사+동사"로 강하게 박힌 스냅샷이어야 한다.

나쁜 예 (설명형):
- "물안개 속에서 / 혼자 걷는 이 밤" (상황 설명)
- "지워지지 않는 이 감정들이" (감정 메타)

좋은 예 (스냅샷형):
- "가로등이 번져" (명사+동사)
- "발자국이 미끄러져" (명사+동사)
- "유리 위에 숨이 남아" (명사+동사)

배치 규칙:
- 스냅샷 훅: 1~2행에 배치
- 정서 문장: 3~4행에 배치

목적:
- 훅의 중독성과 인지 속도 향상
- 10초 내 훅 인지 QC 통과율 향상

---

### 1.7 Bridge Thesis Constraint (테제 제약)

- Bridge Anchor의 Thesis(고정 1라인)는 **사람/관계 의존**이 아닌 **현상 기반**이어야 한다.

나쁜 예 (사람 의존):
- "너의 낮은 목소리"
- "네가 남긴 흔적"
- "우리의 그림자"

좋은 예 (현상 기반):
- "젖은 밤의 흔적"
- "번져버린 발자국"
- "유리 위의 숨"
- "사라지지 않는 습기"

목적:
- 트랙 간 테제 겹침 방지
- AI가 "사람 레퍼런스"로 과잉 해석하는 것 차단

---

### 1.8 V2 Escalation Rule (2절 상승)

- Verse 2의 **마지막 2행**은 감정 에스컬레이션을 유도해야 한다.
- 단순 동일 구조가 아닌, **고음/호흡 강도를 자극하는 단어** 선택

유도 방법:
1. **동사 강화**: 약한 동사 → 강한 동사 (스쳐가 → 휩쓸려)
2. **감각 강화**: 시각 → 촉각/체온 (희미해 → 차가워)
3. **호흡 단어**: 숨, 목, 가슴, 떨림 등 신체 반응 단어 추가

예시:
- V1 마지막 2행: "걸음을 멈춰도 / 소리만 맴돌아"
- V2 마지막 2행: "숨을 삼켜봐도 / 가슴이 떨려와" (← 신체 반응 추가)

목적:
- Suno가 V2 후반부를 더 강하게 부르도록 유도
- 감정선 상승 없이 평탄해지는 것 방지

---

### 1.9 Physical Object Anchor Rule (물성 오브젝트 앵커) — NEW

> **"추상어 과밀"은 무난함의 주범이다.**

- 각 섹션(Verse, Bridge, Chorus)에 **최소 1개의 물성 있는 오브젝트**를 배치한다.
- 추상어(윤곽, 경계, 흐려, 지워, 사라져 등)만으로 채우면 **이미지가 안 잡히고 감정선이 평평해진다**.

**물성 오브젝트 예시 (도시/밤 컨셉):**
- 네온, 표지판, 횡단보도, 유리문
- 지하철 창, 엘리베이터 거울, 가로등
- 코트, 우산, 담배 연기, 커피잔

**나쁜 예 (추상어 과밀):**
```
형체가 흐려져
희미한 불빛 속에
경계가 지워져
그 속에 서 있어
```
→ 모든 줄이 같은 질감의 추상어 → 장면이 안 잡힘

**좋은 예 (물성 1개 추가):**
```
형체가 흐려져
네온의 번짐 속에  ← 물성 (네온)
경계가 지워져
그 속에서 얼어
```
→ "네온"이라는 물성이 장면을 고정시킴

**적용 규칙:**
- Verse: 최소 1개 물성 오브젝트
- Bridge: Scene 라인에 동작/감각 + 물성
- Chorus: 훅 라인에 장면+동사 (1.6 참조)

**FAIL 조건:**
- 4행 전체가 추상어로만 구성됨 → FAIL
- 물성 오브젝트 없이 컨셉 단어만 반복 → FAIL

---

### 1.10 Image Density Management (이미지 과밀 방지) — v1.8 NEW

> **"이미지가 너무 풍부하면 서로 발목을 잡는다."**

V1과 V2에 같은 이미지(창가, 빛, 먼지, 온기, 시간 등)가 반복 등장하면:
- 곡이 흐르지 않고 정체된 느낌
- "drifting" 감각이 줄어듦
- 플레이리스트 반복 청취 시 피로도 증가

**해결 원칙:**
```
V1 = 공간 (Space) — 물리적 배경, 위치, 오브젝트
V2 = 감각 (Sensation) — 신체 반응, 시간 흐름, 내면 체감
```

**나쁜 예 (이미지 과밀):**
```
[V1] 창가에 빛이 쏟아져 / 먼지가 떠다녀
[V2] 창가에 시간이 내려앉아 / 온기가 스며들어
```
→ "창가" 반복, 이미지 축 동일

**좋은 예 (축 분리):**
```
[V1] 창가에 빛이 쏟아져 / 먼지가 떠다녀  ← 공간
[V2] 바닥에 시간이 내려앉아 / 숨결이 스며들어  ← 감각
```
→ V1은 눈에 보이는 장면, V2는 몸으로 느끼는 감각

**FAIL 조건:**
- V1과 V2에서 핵심 이미지 단어가 2개 이상 겹침 → 재작성

---

### 1.11 Chorus Tone Rule (후렴 관조 톤) — v1.8 NEW

> **"플레이리스트는 반복 청취된다. 직접 호소는 피로해진다."**

Chorus에서 직접적인 명령/호소 표현은 1회 청취에는 강렬하지만,
플레이리스트 반복 청취 시 과잉으로 느껴진다.

**원칙:**
- 직접 호소(~아, ~야) → 관조 톤(~은, ~는)으로 전환
- 명령형 2개 이상 연속 금지

**나쁜 예 (직접 호소 과다):**
```
이 순간 여기 머물러
시간아 천천히 가
```
→ "머물러" + "시간아" = 명령 2연속

**좋은 예 (관조 톤):**
```
이 순간 여기 머물러
시간은 조용히 가
```
→ 1개 명령 + 1개 관조 = 균형

**적용 규칙:**
- Chorus 4행 중 직접 호소/명령형은 **최대 1개**
- 나머지는 관조/묘사 톤 유지

## 🧼 2. Pure Lyric Input Rule (입력 무결성)

### 2.0 VIBE-M이 가이드와 다르게 가는 이유 — v1.6 NEW

> **"가이드에서 권장하는 괄호/메타 지시는 Suno에서 가사로 낭독될 확률이 높아 VIBE-M에서는 금지한다 (순수 가창 데이터 보존 목적)."**

**배경:**
- 일부 Suno 가이드는 가사 입력란에 `(female)`, `(Scene: ...)` 같은 메타 정보를 넣으라고 권장
- 그러나 실제 테스트 결과, Suno가 이를 **가사로 읽어버리는 경우가 빈번**
- VIBE-M은 이 문제를 Case 04로 기록하고, 순수 가창 데이터만 입력하는 방식을 채택

**VIBE-M 원칙:**
| 항목 | 가이드 권장 | VIBE-M 방식 | 이유 |
|------|-----------|-------------|------|
| 보컬 성별 | `(female)` 가사에 삽입 | Style Prompt에 명시 | 낭독 방지 |
| 씬 설명 | `(Scene: rainy night)` | Style Prompt에 명시 | 낭독 방지 |
| 감정 지시 | `(Emotion: sad)` | Style Prompt에 명시 | 낭독 방지 |
| Performance Cue | `(whispered)` | **허용** (1단어만) | 동작 확인됨 |

**결론:**
- 가사 입력란 = **순수 가창 텍스트 + 구조 태그 + Performance Cues**
- 모든 스타일/연출 지시 = **Style Prompt로 이동**

---

### 2.1 절대 금지 사항

Suno 가사 입력란에 다음을 **절대 넣지 않는다**.

- 영어 설명 문장
- **설명형 괄호 지시어**: `(Scene: ...)`, `(Emotion: ...)`, `(Mood: ...)`
  - 이유: Suno가 가사로 읽어버림 (Case 04)
- Scene 설명
- 악기/편곡 지시
- 감정 메타 코멘트
- **(female)**, **(male)** 같은 보컬 성별 지시 — Style Prompt로 이동

이 모든 것은 STYLE.md로 이동한다.

> **참고**: 짧은 보컬 연출 지시어 `(whispered)`, `(belted)` 등은 2.3 Performance Cues 참조

---

### 2.2 허용되는 태그 — v1.9 태그 사전 고정

> **VIBE-M은 태그 최소화가 원칙. 허용 태그 외 사용 금지.**

#### 🚫 금지 태그 (FORBIDDEN) — Lyrics에서 절대 사용 금지

> **편곡/악기 지시는 반드시 Style Prompt로 이동**

| 금지 태그 예시 | 이유 | 대체 위치 |
|---------------|------|----------|
| `[Kick in]`, `[Drums enter]` | 악기 진입 지시 | Style Prompt |
| `[Groove locks in]` | 편곡 지시 | Style Prompt |
| `[Subtle lift]`, `[Build up]` | 다이내믹 지시 | Style Prompt |
| `[Pad widens]`, `[Bass enters]` | 악기 변화 지시 | Style Prompt |
| `[Vibraphone solo]` | 악기 솔로 지시 | Style Prompt |

**원칙:**
- Lyrics = **보컬이 할 일만** (발성, 화음, 스타일)
- Style Prompt = **악기/편곡이 할 일** (진입, 밀도, 다이내믹)

---

**🔴 필수 (REQUIRED) — 모든 트랙에 사용:**
```
[intro], [outro]
[verse], [verse1], [verse2]
[bridge], [bridge1], [bridge2]
[chorus]
[instrumental]
```

**🟡 조건부 허용 (OPTIONAL) — 필요할 때만:**

| 태그 | 사용 조건 | 예시 상황 |
|------|----------|----------|
| `[pre-chorus]` | Hook 진입 타이밍 제어 필요 시 | K-POP 스타일 빌드업 |
| `[hook]` | 코러스와 별개 훅 강조 필요 시 | 힙합/댄스 장르 |
| `[breakdown]` | 긴 인스트루멘탈 필요 시 | EDM/일렉트로닉 |
| `[end]` | 명확한 종료 필요 시 | Outro가 길어질 때 |
| `[big finish]` | 마지막 코러스 화려하게 | 발라드 클라이맥스 |

**🟢 자유 사용 (FREESTYLE) — 상황에 따라:**
- `[break]` - 짧은 휴식/빌드업
- `[interlude]` - 분위기 전환 구간
- `[refrain]` - 살짝 다른 스타일로 부르기
- `[drop]` - 드롭 (EDM 스타일)
- `[sting]` - 섹션 간 짧은 음악적 브레이크

**⚠️ 주의 사용 (USE WITH CAUTION):**

> 아래 태그들은 Style Prompt와 중복될 수 있음. 둘 다 쓰면 충돌 위험.

- `[Sad Verse]` - 멜랑콜리한 딜리버리
- `[Happy Chorus]` - 밝고 업리프팅
- `[Quiet arrangement]` - 미니멀, 친밀한 사운드
- `[Slow Tempo]` - 인스트루멘탈 구간 느리게
- `[Sung with Emotion]` - 감정적 보컬

**🔵 고급 태그 (ADVANCED) — 실험적:**

> Style Prompt가 제대로 동작하지 않을 때 대안으로 시도

- `[Vocal Style: soulful, husky]` - 보컬 스타일 직접 지정
- `[Dynamics: soft verses, powerful chorus]` - 강약 조절
- `[Mood: dreamy, melancholic]` - 분위기 지정
- `[Atmosphere: spacious, ambient]` - 공간감 지정

**보컬 제어 메타태그 (v1.4):**

> 구조 태그와 함께 사용하여 보컬 스타일 제어

| 태그 | 효과 | 사용 예시 |
|------|------|----------|
| `[No harmony]` | 화음/코러스 제거 | `[verse1]` 바로 뒤에 |
| `[No backing vocals]` | 백보컬 제거 | 노래 시작 부분 |
| `[Direct vocal]` | 단단한 보컬 강조 | |
| `[Dry voice]` | 건조한 보컬 | |
| `[Chest voice]` | 진성 강조 | |
| `[No falsetto]` | 가성 제거 | |
| `[Powerful belt]` | 강한 벨팅 | Chorus에 |
| `[Soft belt]` | 부드러운 벨팅 | Chorus에 |

**사용 예시:**
```
[verse1]
[Direct voice, No harmony]
가사 내용...

[chorus]
[Powerful belt, Chest voice]
가사 내용...
```

**주의:** 메타태그는 구조 태그 바로 뒤 별도 행에 배치. 가사와 같은 행에 쓰지 않음.

### 2.3 Performance Cues (인라인 연출) — v1.5 NEW

> 구조 태그 바로 뒤, 가사 전에 배치하여 보컬 연출 지시

**허용되는 Performance Cues (괄호 형식):**
- `(whispered)` - 속삭임
- `(soft)` - 부드럽게
- `(belted)` - 벨팅
- `(airy)` - 공기감 있게
- `(breathy)` - 숨결 섞인
- `(high)` - 고음역
- `(half-time)` - 하프타임 느낌

**사용 예시:**
```
[verse1]
(soft)
문틈 사이로
빛줄기 스며와

[chorus]
(belted)
문틈 사이로
빛이 번져와
```

**주의사항:**
- 영어 단어 **1개만** 사용 (설명 금지)
- 구조 태그 바로 뒤에 배치
- 가사와 같은 행에 쓰지 않음
- `(Scene: ...)` 같은 설명형은 **절대 금지** (Case 04 참조)

### 2.4 가사 길이 가이드라인 — v1.5 NEW

> 짧고 일관된 가사가 Suno 결과의 안정성을 높인다

**권장 길이:**
- 전체: **100-120 단어**
- 섹션당: **4-6행**
- 행당: **6-10음절** (한국어)

**이유:**
- 과도하게 긴 가사 → Suno가 일부 무시하거나 압축
- 짧고 반복적 → 멜로디 일관성 향상
- Case 12 참조: Chorus 6행 → 피로감 발생

## 🚫 3. Failure Case Archive (대표적인 실패 유형)

### Case 01: Metric 붕괴
- Verse 2에서 음절 수 증가/감소
- 결과: 멜로디 붕괴, AI 즉흥 변주 발생

### Case 02: Ending 불일치
- V1은 명사, V2는 의문문
- 결과: 후반 멜로디가 다른 곡처럼 들림

### Case 03: Bridge 설명조
- "~한 마음", "~한 감정"
- 결과: AI 특유의 내레이션 톤 발생

### Case 04: Meta Instruction Hallucination
- `(Scene: Wet breath)` 같은 문구 삽입
- 결과: 보컬이 이를 가사처럼 읽어버림

### Case 05: Vocabulary Overlap (키워드 중복)
- Track 03 제목이 "잔향"인데 Track 04에서 "잔향만 맴돌아" 사용
- 결과: 알고리즘이 "같은 곡 반복"으로 오인, 플레이리스트 다양성 저하

### Case 06: Descriptive Hook (설명형 훅)
- "물안개 속에서 / 혼자 걷는 이 밤" (상황 설명)
- 결과: 훅 인지 속도 저하, 중독성 부족

### Case 07: Person-Dependent Thesis (사람 의존 테제)
- Bridge Thesis가 "너의 낮은 목소리"
- 결과: 트랙 간 테제 겹침, AI 과잉 해석 (사람 레퍼런스 증폭)

### Case 08: Flat V2 (V2 에스컬레이션 부재)
- V1과 V2 마지막 2행이 감정 강도 동일
- 결과: 곡 전체가 평탄하게 느껴짐, Suno가 V2를 약하게 부름

### Case 09: Abstract Word Density (추상어 과밀) — NEW
- 4행 전체가 추상어(윤곽/경계/흐려/지워/사라져)로만 구성
- 물성 오브젝트 없이 컨셉 단어만 반복
- 결과: 이미지가 안 잡힘, "AI스러운 문장", 무난함 발생

### Case 10: Ending 품사 불일치 — NEW
- V1 마지막 행이 동사인데 V2 마지막 행이 형용사
- 예: "서 있어"(동사) vs "차가워"(형용사)
- 결과: 멜로디 끝 처리가 다르게 재현됨, 곡의 일관성 저하

### Case 11: V1-V2 행 수 불일치 (Track 06 교정) — NEW
- V1은 4행인데 V2를 6행으로 "다른 느낌"을 주려고 시도
- 예: V1 (4행, 22음절) vs V2 (6행, 36음절)
- 결과: Suno가 V2를 V1 멜로디에 억지로 얹으려다 멜로디 붕괴 100%
- **교훈: "다른 컨셉"은 행 수가 아니라 단어 선택으로 표현**

### Case 12: Chorus 과다 행 (Track 06 교정) — NEW
- Chorus를 6행으로 작성 (규칙: 3~4행)
- 결과: 훅이 너무 길어 리스너 피로감, 반복 시 Monotony 발생
- **교훈: Chorus는 무조건 3~4행. 길면 압축.**

### Case 13: Bridge Thesis 변경 + 키워드 충돌 (Track 06 교정) — NEW
- Bridge2 Thesis를 B1과 다르게 변경 ("적막이 더 깊어져" → "사라지는 윤곽뿐")
- "윤곽"은 Track 02 제목 → 1.5 Vocabulary Independence 위반
- 결과: 곡의 일관성 저하 + 플레이리스트 키워드 중복
- **교훈: Bridge Thesis는 B1=B2 동일. 다른 트랙 제목 키워드 절대 금지.**

## 🧩 4. Canonical Structure (표준 구조)

### 4.1 VIBE-M 기본 구조

모든 VIBE-M 트랙은 아래 구조를 기본으로 한다.

```
[intro]
[verse1]
[bridge1]
[chorus]
[verse2]
[bridge2]
[chorus]
[chorus]
[outro]
```

### 4.2 구조 공식 옵션 — v1.5 NEW

> 장르/컨셉에 따라 다른 구조 선택 가능

| 구조 | 공식 | 용도 | 대표 예시 |
|------|------|------|----------|
| **VIBE-M Default** | V-B-C-V-B-C-C | 감성 R&B, 브릿지 강조 | 기본 |
| **Pop Standard** | V-C-V-C-B-C | 가장 대중적, 안정감 | 아이유 밤편지 |
| **K-POP Standard** | V-P-C-V-P-C-B-C | Pre-Chorus로 고조 | 블랙핑크 |
| **Storyteller** | V-V-C-V-C | 스토리텔링 중심 | 빌리진 |
| **Short & Sweet** | C-V-C-V-C | 짧고 강렬, 힙합 | Still Dre |

**태그 변환:**
- V = `[verse]`
- C = `[chorus]`
- B = `[bridge]`
- P = `[pre-chorus]`

**선택 기준:**
- 감성/R&B → VIBE-M Default 또는 Pop Standard
- K-POP/댄스 → K-POP Standard
- 힙합 → Short & Sweet
- 포크/발라드 → Storyteller
